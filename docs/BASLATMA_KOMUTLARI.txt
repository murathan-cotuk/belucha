GIT WORKFLOW
============

DOĞRU ÇALIŞMA SIRASI:
----------------------

1. DEV BRANCH'İNDE ÇALIŞ:
   git checkout dev
   git add .
   git commit -m "feat: yeni özellik"
   git push origin dev

2. TEST ET (dev branch'inde):
   npm run dev
   # Her şey çalışıyor mu kontrol et

3. MAIN'E MERGE ET (sadece test başarılıysa):
   git checkout main
   git merge dev
   git push origin main

⚠️  ÖNEMLİ:
   - Dev branch = Development (geliştirme)
   - Main branch = Production (canlı)
   - Main'e sadece test edilmiş, çalışan kodlar gider!

VERCEL DEPLOYMENT SORUSU
=========================

SORU: Medusa'yı Vercel'e ayrı mı deploy edeyim?

CEVAP: HAYIR, Medusa'yı Vercel'e deploy ETMEYİN. İşte nedenleri:

❌ VERCEL'E DEPLOY ETMEYİN ÇÜNKÜ:

1. VERCEL SERVERLESS:
   - Vercel serverless functions kullanır
   - Medusa backend sürekli çalışan bir servis olmalı
   - Serverless Medusa için uygun değil

2. DATABASE BAĞLANTISI:
   - Medusa PostgreSQL veya başka bir database kullanır
   - Vercel'de database bağlantıları sorunlu olabilir
   - SQLite Vercel'de çalışmaz

3. REDIS GEREKSİNİMİ:
   - Medusa Redis kullanır (cache, queue)
   - Vercel'de Redis kurulumu zor

✅ ÖNERİLEN DEPLOYMENT:

MEDUSA BACKEND İÇİN:
- Railway.app (railway.app) - ÖNERİLEN
